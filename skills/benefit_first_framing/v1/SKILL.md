# benefit_first_framing

## 策略描述

在标题和描述的最前端，直接呈现用户最具体、最可感知的收益或结果，将品牌名称、功能描述、产品特性等信息后置。核心逻辑是：当用户对品牌认知度低时，"你能得到什么"比"这是什么"更能触发点击动机。该策略绕过好奇心驱动机制，转而以显性价值承诺作为点击锚点，让用户在0.3秒的扫描窗口内即可判断内容与自身需求的相关性。

**策略背景：** `curiosity_gap` 在低品牌认知场景（如 'meetspot'）中 CTR 提升为 0.0000，表明悬念式表达在用户缺乏品牌信任基础时无法有效转化。`benefit_first_framing` 通过明确的价值前置，替代依赖好奇心的隐性吸引逻辑。

---

## 适用场景

- **低品牌认知度页面**：用户首次接触品牌，无法凭品牌名产生信任或兴趣
- **竞争激烈的通用关键词**：搜索结果页同质化严重，需要在前3个词内完成差异化
- **工具类 / 效率类产品**：用户带着明确任务意图搜索，直接告知结果比制造悬念更有效
- **本地化服务页面**：用户需要快速确认服务是否覆盖其需求（地点、场景、人群）
- **转化漏斗中段页面**：用户已了解品类，正在对比选项，具体收益比品牌故事更能推动决策

**不适用场景：**
- 品牌词搜索（用户已有明确意图，品牌名本身即为锚点）
- 内容型 / 媒体型页面（读者预期叙事性表达）

## 核心技巧

1. **收益动词开头原则**
   标题首词或前两词使用可感知动作的动词，如"找到"、"预订"、"节省"、"获取"、"解决"，直接指向用户行为结果，而非描述产品功能。

2. **具体化优于抽象化**
   将模糊收益替换为可量化或可场景化的表达：
   - ❌ "更好的会议体验" → ✅ "10分钟内找到附近可用会议室"
   - ❌ "专业服务" → ✅ "当天可预订，无需押金"

3. **用户视角替代产品视角**
   将"我们提供X功能"改写为"你可以实现Y结果"，主语从品牌切换到用户。

4. **收益前置 + 品牌/特性后置结构**
   推荐标题结构：`[具体收益] - [品牌名/产品名]`
   推荐描述结构：`[核心收益句]. [支撑细节]. [行动召唤].`

5. **匹配搜索意图的收益层级**
   - 信息型意图：收益 = 获得清晰答案 / 节省研究时间
   - 导航型意图：收益 = 直达目标 / 减少跳转
