# intent_match_confirmation

## 策略描述

直接确认用户搜索意图的标题与描述优化策略。通过在标题和描述的最前端明确点名用户所搜索的产品、服务或解决方案，配合其最清晰的价值主张，消除任何歧义，对导航型和品牌型查询给予即时相关性信号。

**核心心理机制：** 与好奇心缺口策略（通过信息缺失制造悬念）相反，本策略采用「信息前置」机制——用户看到结果的第一眼即获得「这正是我要找的」的确认感，从而降低认知摩擦，提升点击决策速度。

**适用查询类型：** 品牌词查询、导航型查询、产品名称查询、工具类查询。这类查询的用户已有明确目标，不需要被吊起好奇心，而是需要被「认出来」。

---

## 适用场景

- **品牌/导航型查询**：用户直接搜索品牌名或产品名（如「meetspot」「Notion AI」「Figma 协作」）
- **功能确认型查询**：用户搜索某一具体功能或工具（如「在线甘特图工具」「PDF 转 Word 免费」）
- **竞品比较前的品牌查询**：用户在决策漏斗中段，已知品牌名但需确认其核心价值
- **复访用户的再触达**：曾经接触过品牌、通过品牌词回来验证的用户
- **本地/垂直服务查询**：搜索特定服务类别时附带明确限定词（如「上海企业注册代理」）

**不适用场景：** 泛类目探索型查询（如「如何提高工作效率」）、内容消费型查询（如「最好的播客推荐」）——这类场景更适合好奇心缺口策略。

## 核心技巧

**1. 品牌/产品名前置（Title 首词原则）**
将用户搜索的核心词作为标题第一个词或第一个短语，让搜索引擎加粗匹配词的同时，让用户眼睛第一落点即命中目标词。

**2. 价值主张即时锚定**
在确认产品名之后，立刻用最简洁的语言说明「它能为你做什么」，格式参考：`[产品名] — [一句话核心价值]`，避免形容词堆砌，优先使用动词或结果导向的表达。

**3. 描述段首句复述意图**
Description 的第一句话应当再次确认用户的搜索意图，充当「是的，你找对了」的角色，第二句话再展开功能亮点或行动召唤。

**4. 消除歧义限定词**
当品牌名或产品名可能有多重解读时，加入一个精准限定词（行业词、场景词、用户角色词）消除歧义，避免非目标用户点击浪费预算，同时提升目标用户的确认感。
