# direct_value

## 策略描述

通过明确陈述产品或服务的核心功能、特性或利益来优化标题和描述，使用清晰、功能性的语言匹配用户已确定的搜索意图。

当用户搜索品牌词或导航类关键词时，他们已经知道自己想要什么——不存在信息不对称，悬念式写法（curiosity_gap）无法制造有效吸引力。direct_value 策略用具体的功能亮点、使用场景或行动导向的描述替代模糊的预告式语言，通过确认用户意图来强化点击决策，而非试图制造人为悬念。

## 适用场景

- **品牌词 / 导航型查询**：用户已知目标，搜索"meetspot"、"Notion"、"Figma"等品牌名
- **功能型关键词**：用户搜索具体功能，如"在线会议室预订"、"团队日程协调工具"
- **高购买意图查询**：用户处于决策末期，需要的是确认而非激发好奇
- **竞品对比场景**：用户在比较选项，需要清晰的差异化价值点
- **返回用户召回**：老用户重新搜索，需要功能更新或核心价值的直接提示

**不适用场景**：内容发现型、娱乐型或用户尚未明确需求的探索性查询（此类场景更适合 curiosity_gap）

## 核心技巧

1. **功能前置**：将最核心的功能或使用场景放在标题最前端，不要用修饰词开头
2. **数字与具体性**：用具体数字、时间、规模替代模糊形容词（"3步完成预订"优于"快速完成预订"）
3. **动词驱动**：用动宾结构明确用户能做什么（"预订 · 协调 · 确认"），而非描述产品是什么
4. **意图镜像**：描述语言应与用户搜索意图的措辞高度匹配，直接呼应其已有认知
5. **消除摩擦点**：在描述中主动解答用户最可能的顾虑（免费、无需注册、即时可用）
6. **特性枚举**：用短横线或中间点分隔3个以内的核心特性，提升可扫描性

## 示例

### 示例 1：品牌导航词优化

**关键词**：meetspot

❌ Before（curiosity_gap 风格）
> 标题：你不知道的会面方式，正在改变团队协作
> 描述：当传统会议室预订遇上新问题，有一种方案让所有人都说"就是这个"……

✅ After（direct_value 风格）
> 标题：Meetspot · 团队会议室预订与场地协调平台
> 描述：在线预订会议室、同步团队日程、实时确认场地可用性。支持多地点管理，免费注册即用。
