# brand_value_reinforcement

## 策略描述

针对导航型和品牌型查询，通过在标题和描述中显著突出品牌名称并直接陈述其核心价值主张或主要使用场景，强化用户认知并确认页面相关性。该策略摒弃好奇心驱动或情感煽动式文案，转而采用信任强化与品牌认同的写作逻辑——用户已知晓目标品牌，所需的不是吸引，而是确认。

**核心逻辑：** 品牌型查询的点击决策由"相关性确认"驱动，而非情感刺激。领先展示品牌名称 + 简洁的功能定义，是影响此类查询 CTR 的真正杠杆。

## 适用场景

- 用户直接搜索品牌名称（如 `meetspot`、`notion`、`figma`）
- 搜索词为品牌名 + 功能词（如 `meetspot 会议`、`meetspot scheduling`）
- 品牌首页、登录页、核心产品页的 SEO 优化
- 用户具有明确导航意图，已决定访问该品牌，仅需确认入口正确
- 竞品品牌词防御场景（需快速建立辨识度）

**不适用场景：**
- 信息型查询（用户尚未决定品牌选择）
- 通用关键词竞争（需要差异化吸引力）
- 内容营销页面（需要内容预告与点击诱因）

## 核心技巧

1. **品牌名前置原则**
   标题首词或首段即出现品牌名，避免将品牌名置于标题末尾或省略。格式推荐：`{品牌名} – {核心功能定语}` 或 `{品牌名} | {使用场景描述}`。

2. **价值主张一句话定义**
   用最短的语言回答"这是什么"，避免形容词堆砌和夸张修辞。优先使用名词短语而非动词短语（"团队会议调度平台"优于"帮你轻松搞定会议安排"）。

3. **描述中重复品牌名**
   在 meta description 的首句再次出现品牌名，强化双重确认信号，同时覆盖搜索引擎对品牌词的加粗高亮展示。

4. **功能词精准匹配**
   将品牌的核心使用场景以名词形式嵌入标题（如 `scheduling`、`project management`、`design tool`），与用户搜索词形成精准呼应，而非模糊的品类描述。

5. **避免情感化与悬念化表达**
   删除所有好奇心缺口结构（"你不知道的…"、"改变你…的方式"）、感叹号、以及无实质内容的形容词（"强大的"、"革命性的"、"终极"）。
