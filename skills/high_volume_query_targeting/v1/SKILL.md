# high_volume_query_targeting

## 策略描述

识别并优先处理 Google Search Console 中月均展示量 ≥ 500 次的 URL，确保每一次标题/描述优化实验都建立在充足的流量基础之上，从而产生统计上可测量的 CTR 提升数据。

**核心逻辑：** 过往实验失败的根本原因是在低流量查询上运行测试，导致 CTR 提升为 0.0000 且状态为 `insufficient_data`。本策略通过强制设置 500+ 月均展示量门槛，确保任何心理学文案钩子（好奇心缺口、数字钩子、损失厌恶）或结构性标题变更都有足够的基线流量来产生有意义的实验结论。

**决策流程：**
1. 从 GSC 导出过去 28 天数据，筛选展示量 ≥ 500 的 URL
2. 按 CTR 升序排列，优先处理 CTR 偏低但展示量充足的页面（高潜力标的）
3. 选择适配的心理学钩子并改写标题/描述
4. 记录实验前基线 CTR，设置 28 天观察周期
5. 对比实验组与对照组数据，判断是否达到统计显著性

---

## 适用场景

- **强制适用条件：** 目标 URL 过去 30 天 GSC 展示量 ≥ 500 次
- **优先级最高场景：** 展示量 ≥ 500 且当前 CTR < 行业均值（信息类页面 < 3%，电商类 < 2%）
- **次优场景：** 展示量 500–1000，CTR 有明显提升空间（相对均值低 30% 以上）
- **禁止实验场景：** 展示量 < 500 的 URL（数据不足，实验结果不可信）
- **暂缓场景：** 近期经历重大算法更新或站点迁移的 URL（基线数据不稳定）

**适用页面类型：**
- 高流量博客文章 / 指南类内容
- 核心产品页 / 分类页
- 工具页 / 资源页
- 任何已验证有搜索需求但点击率偏低的落地页
