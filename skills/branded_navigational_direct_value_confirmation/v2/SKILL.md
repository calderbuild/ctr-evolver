# branded_navigational_direct_value_confirmation

## 策略描述
针对品牌词或导航型查询，放弃悬念式、好奇心驱动的标题框架，转而直接、明确地陈述品牌的核心价值主张。用户在输入品牌名称时，意图已经确定——他们知道自己在找什么，不需要被"钩住"，只需要被"确认"。本策略的核心逻辑是：**消除摩擦，而非制造悬念**。通过清晰表达产品做什么、为什么值得现在点击，与用户已形成的预期精准匹配，从而提升点击率和转化质量。

## 适用场景
- 用户直接搜索品牌名称（如 "Meetspot"、"Notion"、"Figma"）
- 导航型查询：用户目的是找到某个特定网站或产品入口
- 品牌词 + 功能词组合查询（如 "Meetspot scheduling"、"Notion templates"）
- 竞品对比场景中品牌方自身的 SEO 防御性布局
- 老用户回访路径优化（用户已知品牌，需要快速确认入口）

**不适用场景：**
- 信息型查询（如 "how to schedule meetings"）
- 问题导向的长尾词（用户尚未形成品牌认知）
- 探索型查询（用户处于发现阶段）

## 核心技巧

1. **价值确认优先于价值发现**
   不要假设用户不知道品牌是什么。直接陈述品牌最核心的一句话价值，让用户在 0.5 秒内完成"这就是我要找的"判断。

2. **品牌名前置 + 破折号结构**
   使用 `品牌名 — 核心价值陈述` 的格式，品牌名作为锚点，破折号后接最直接的功能描述，避免任何需要"解读"的表达。

3. **消除悬念词汇**
   删除所有暗示"你不知道但应该知道"的词汇，如："发现"、"揭秘"、"你不知道的"、"原来"、"竟然"。这类词在导航型查询中产生认知错位，降低可信度。

4. **行动触发词替代好奇触发词**
   用 "立即"、"免费开始"、"直接"、"无需"、"一键" 等行动导向词，替代 "了解"、"探索"、"发现" 等模糊引导词。

5. **Description 承接标题，补充具体场景**
   Meta Description 不重复标题，而是列举 2-3 个具体使用场景或消除用户顾虑的信息点（如免费、无需注册、支持团队协作），强化"现在就点"的理由。

6. **避免过度承诺与空洞形容词**
   删除 "最好的"、"革命性的"、"颠覆性的" 等无法被用户即时验证的形容词，用具体功能描述替代。
