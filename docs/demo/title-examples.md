# Before/After Title Examples

Three skills applied to the same page and query, demonstrating how different strategies produce fundamentally different titles.

**Target page**: `https://meetspot-irq2.onrender.com/en/`
**Query**: "meetspot"
**Original title**: "MeetSpot - Find the Perfect Meeting Spot"
**Query type**: Branded / Navigational

---

## 1. curiosity_gap (Seed Skill)

**Strategy**: Create an information gap that compels clicks. Hint at valuable information without fully revealing it.

| | Title |
|---|---|
| **Original** | MeetSpot - Find the Perfect Meeting Spot |
| **Generated** | MeetSpot: The Meeting Space Secret Revealed |

**What changed**:
- Added mystery element ("Secret Revealed")
- Implies there's hidden value the user doesn't know about
- Targets emotional curiosity

**Problem**: For a branded query like "meetspot", the user already knows what they want. Creating an information gap adds friction instead of reducing it. The user isn't curious about a secret -- they just want to get to MeetSpot.

---

## 2. intent_match_confirmation (Evolved, Step 1)

**Strategy**: Directly confirm user search intent. Place the searched product name at the front, paired with the clearest value proposition. Opposite of curiosity gap -- uses "information frontloading" for immediate relevance.

| | Title |
|---|---|
| **Original** | MeetSpot - Find the Perfect Meeting Spot |
| **Hypothetical** | MeetSpot -- Team Meeting Scheduling Platform |

**What changed**:
- Brand name leads (first-word principle)
- Functional definition immediately follows ("Team Meeting Scheduling Platform")
- Zero mystery, zero friction
- User's first glance confirms: "this is exactly what I'm looking for"

**Key insight**: This skill was generated by the evolution system after analyzing failures of curiosity_gap on branded queries. The system independently discovered that navigational intent requires confirmation, not curiosity.

---

## 3. brand_value_reinforcement (Evolved, Step 2)

**Strategy**: For branded/navigational queries, prominently display brand name + core value proposition. Reject emotional or curiosity-driven copy. Trust reinforcement over attention-grabbing.

| | Title |
|---|---|
| **Original** | MeetSpot - Find the Perfect Meeting Spot |
| **Hypothetical** | MeetSpot | Smart Meeting Scheduling for Teams |

**What changed**:
- Clean separator format (`Brand | Value`)
- Noun-phrase value proposition ("Smart Meeting Scheduling for Teams")
- No adjective piling, no exclamation marks, no suspense
- Description would repeat brand name in first sentence for double confirmation

**Core logic from SKILL.md**: "Brand query click decisions are driven by relevance confirmation, not emotional stimulation. Leading with brand name + concise functional definition is the true lever for this query type's CTR."

---

## Pattern: Evolution Discovers Strategy Fit

```
Seed strategy:        curiosity_gap  →  "The Secret Revealed"
                                         ↓ fails on branded queries
Evolution step 1:     intent_match   →  "Team Meeting Scheduling Platform"
                                         ↓ further specialization
Evolution step 2:     brand_value    →  "Smart Meeting Scheduling for Teams"
```

The system moved from a generic attention-grabbing strategy to domain-appropriate intent confirmation -- a genuine SEO insight that professional SEOs would recognize as correct for branded queries.
